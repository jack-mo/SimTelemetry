About

SimTelemetry is an open source application providing telemetry from various racing or driving simulators. Development silentely started in mid 2011 by Hans de Jong in the aim of providing a purpose for it's secondary screen, not used during racing. Initial versions started out only for rFactor v1.255, but quickly a plug-in system was implemented so various simulators could be supported.

Currently SimTelemetry has support for 'only' 3 simulators: rFactor 1 (V1.255), rFactor 2 (build 61) and GTR2.

Once core development has stabilised the aim is primiraly focus on getting support for a wider range of simulators, and also deeper for support for existing ones. I will highly appreciate any one who can make contributions to either the core program or simulator plug-ins.

Features

SimTelemetry aims to be an all-in-one for telemetry needs. It can live display and record telemetry for further analysis. In contrast to some other game plug-ins, which overlay timing screens, traclmaps and car information on the gaming screen, SimTelemetry can only work on secondary monitors or secondary computers/laptops connected via network.

This can be a preference of the user. Personally I don't like the main monitor to be clutted full with gauges, timing sheets, and other icons. This is all seperated out so a complete view is maintained of the track.

At this moment, Simtelemetry has support for the following items:

 
Simulators
  • rFactor v1.255 (full)
  • rFactor 2 build 61 (limited)
  • GTR2 v1.1.0.0 (limited)
Widgets
  • Live trackmap
  • Timing displays with sector and lap time information
  • Speedo/tachometer display
  • Fuel consumption and engine wear estimations
  • Tyre/brake temperature and wear graphic
Telemetry
  • Efficient Telemetry logging of 200+ car parameters
  • Analysis tool for every hot lap
  • Soon more...
